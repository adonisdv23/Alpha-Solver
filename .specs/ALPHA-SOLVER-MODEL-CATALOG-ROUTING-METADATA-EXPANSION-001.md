# ALPHA-SOLVER-MODEL-CATALOG-ROUTING-METADATA-EXPANSION-001

## Purpose
Expand static model catalog metadata and deterministic route preview reporting for product-foundation routing work.

## Scope
- Add explicit routing metadata for backend type, model family, availability status, provider-key requirements, locality, smoke eligibility, cost, latency, context, privacy, capabilities, route tags, tool compatibility, best/avoid guidance, fallback eligibility, operator notes, and review timestamps.
- Keep route preview metadata-only and deterministic.
- Surface selected model metadata, fallback-eligible alternatives, no-call evidence, confidence label, and operator caveat.

## Boundary
This contract does not authorize runtime behavior, `/v1/solve`, provider calls, hosted/local model execution, Ollama calls, tools, browsing, Sheets mutation, A/B key inspection, source-map inspection, unblinding, score interpretation, or readiness/value/superiority claims.

## Evidence rule
Catalog inclusion and route preview metadata are not model quality evidence, provider quality evidence, benchmark evidence, readiness evidence, or Alpha-superiority evidence.
