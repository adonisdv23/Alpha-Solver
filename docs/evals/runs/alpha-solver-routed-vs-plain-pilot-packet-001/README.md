# ALPHA-SOLVER-ROUTED-VS-PLAIN-PILOT-PACKET-001

Docs-and-fixtures-only pilot packet for a later routed-vs-plain evaluation. The purpose is to let an operator later test whether routing behavior adds value beyond a plain single-model answer by comparing route discipline, model/tool recommendation fit, constraints, warnings, fallback, evidence boundaries, confidence or answerability notes, and next action clarity.

## Files

- `objective.md` - pilot objective and route-value focus.
- `task-set.md` - 12 static task cards.
- `route-value-rubric.md` - 0 to 3 pilot scoring rubric.
- `plain-model-comparison-protocol.md` - later comparison protocol.
- `result-capture-template.md` - blank capture template.
- `operator-runbook.md` - safe future runbook.
- `stop-conditions.md` - stop rules.
- `non-actions.md` - work explicitly not done.
- `non-claims.md` - claims not supported.
- `checks-run.md` - validation record for this lane.
- `selected-next-state.md` - post-lane selected next state.
- `selected-next-action.md` - operator review action.

## Protocol boundaries

This packet does not execute the pilot. It does not call providers, run hosted models, run local models, execute tools, browse the web, generate Alpha outputs, generate baseline outputs, score outputs, change scores, unblind, inspect raw Alpha outputs, inspect raw baseline outputs, perform source-map work, mutate Google Sheets, add dependencies, expose `/v1/solve`, or expose dashboard or public API behavior. It makes no readiness, benchmark, production, public, security/privacy, provider, local-model, tool-quality, or Alpha-superiority claims.
