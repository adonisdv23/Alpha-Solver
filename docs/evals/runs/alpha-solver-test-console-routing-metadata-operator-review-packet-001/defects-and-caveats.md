# Defects and Caveats

## Defect categories

- Missing required field.
- Ambiguous preview-vs-execution language.
- Weak or absent evidence-boundary caveat.
- Tool preview appears authorized or executed when it is not.
- Fail-closed route preview is missing or confusing.
- Fallback, warning, or reason grouping is hard to scan.

## Caveats to preserve

- The packet itself is not an operator review result.
- The packet does not run the console.
- Catalog metadata is not quality evidence.
- Route-preview metadata is not provider, local-model, hosted-model, or tool execution evidence.
- Any later full operator product test requires a separate bounded lane.
