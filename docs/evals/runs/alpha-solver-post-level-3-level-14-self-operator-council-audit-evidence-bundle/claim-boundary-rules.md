# Claim-boundary rules

This is evidence for a narrow local-only operator-supervised Self Operator candidate. Do not infer production readiness, hosted readiness, runtime readiness, provider readiness, benchmark superiority, broad MVP readiness, release readiness, or autonomous readiness.

Council seats must distinguish evidence-supported narrow claims from prohibited broad inferences. Any false readiness issue is P0 under `severity-rubric.md`.

## Allowed evidence-support statements

- The repository contains the listed evidence packets.
- PR #486 accepted the limited repeatability review for Council audit evidence-bundle preparation.
- This packet prepares prompts and templates for a future manual Council audit.

## Disallowed claims

Do not claim broad readiness, release status, hosted status, provider status, runtime status, benchmark superiority, autonomous operation, dashboard exposure, `/v1/solve` exposure, deployment, billing, credential validation, or secret access.
