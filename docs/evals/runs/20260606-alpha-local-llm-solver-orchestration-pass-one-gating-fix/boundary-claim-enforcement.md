# Boundary-Claim Enforcement

Pass-one model-produced considerations and assumptions are now checked before normal exposure.

If those fields contain forbidden positive claims about readiness, validation, benchmark standing, provider orchestration, `/v1/solve`, dashboard, production, Alpha superiority, local model quality, evidence-model promotion, or billing, the runner fails closed with the deterministic reason:

`pass_one_boundary_claim_violation_non_evidence`

On that path, the model-produced considerations and assumptions are not returned as normal output fields.

Existing pass-two boundary-claim enforcement remains in place.
