# North-Star Diagnosis

## Diagnosis

Alpha Solver's primary identity is a reasoning/routing plus discrimination/evidence system, not a generic LLM playground. The post-#663 through post-#676 Operator Console sequence is adjacent to that identity when it helps operators inspect artifacts, preserve evidence boundaries, capture receipts, or understand route/expert status. It is not central when it becomes UI polish detached from the reasoning and discrimination jobs.

## Repo-grounded alignment points

### Reasoning/routing

The repository describes Alpha Solver as a reasoning/routing layer with gates, scoring, observability, replay, determinism, and budget guard. Route and expert-preview surfaces align when they make task interpretation, route selection, expert/persona decisions, diagnostics, confidence, and claim boundaries inspectable.

### Discrimination / Value Read

Value Read and discrimination artifacts remain the strongest north-star evidence path because they organize case packets, paired outputs, blinded scoring, score locks, unblinding/final interpretation, and evidence boundaries. This reset does not reopen scoring or interpretation; it preserves that job as a product-direction option.

### Prompt-contract simulation

Prompt-contract simulation and manual no-provider packets demonstrate a pattern for bounded evidence generation without overclaiming runtime or provider behavior. A future workbench could support that pattern if selected by the operator.

### Operator-run capture

Operator-run capture and preflight tools provide a concrete evidence workflow: create or validate case packets, capture outputs, run structural preflights, export packets, and preserve evidence boundaries. Console work aligns when it helps inspect or coordinate this flow without adding unapproved execution.

### Route/expert/SAFE-OUT/confidence inspection

SAFE-OUT, calibrated confidence, route metadata, expert preview, shortlist, and diagnostic concepts are central to the reasoning layer. A product surface that explains those decisions can be aligned if it remains inspection-oriented and claim-safe.

### Budget/receipt/evidence boundaries

Budget gates, receipt concepts, and evidence-boundary copy are useful guardrails. They support Alpha Solver's operating discipline but do not by themselves establish the product job or authorize live execution.
