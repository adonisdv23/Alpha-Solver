# Billing and cost boundary

- No private payment data may be committed.
- The operator must manually verify billing readiness before any API call.
- The first smoke must use a tiny prompt and minimal output.
- Stop if usage is billed unexpectedly.
- Stop if cost metadata is unavailable and the operator requires it.
- Stop if any quota/free-token assumption is unclear.

This packet does not state free-token amounts, model eligibility, account state, billing account state, or payment details.
