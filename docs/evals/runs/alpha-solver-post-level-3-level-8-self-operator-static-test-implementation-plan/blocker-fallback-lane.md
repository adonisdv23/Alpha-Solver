## Use this lane when

Use this fallback if the packet is incomplete, inconsistent, outside the docs-only boundary, missing required files, missing required blocked-behavior coverage, or unable to support future static-test implementation planning.

## Fallback scope

The fallback lane should repair documentation only unless separately approved. It should not implement tests, run Self Operator code, call providers, call external APIs, use credentials, start browser automation, deploy, access billing, or expose routes.
