# Boundary Patterns

## Replace result language with scope language

- Unsafe: "validated quality"
- Safe: "defined the quality-eval boundary"

- Unsafe: "benchmark evidence"
- Safe: "benchmark evidence remains unexecuted and unavailable"

- Unsafe: "Alpha superiority"
- Safe: "comparative superiority is not established without a comparative methodology"

## Replace readiness language with gate language

- Unsafe: "MVP-ready"
- Safe: "MVP readiness requires a separate approved readiness gate"

- Unsafe: "production-ready"
- Safe: "production readiness requires a separate production-readiness packet and evidence"

- Unsafe: "provider-ready"
- Safe: "provider readiness requires provider-specific execution evidence"

- Unsafe: "dashboard-ready"
- Safe: "dashboard readiness requires dashboard-specific validation"

- Unsafe: "`/v1/solve` is ready"
- Safe: "`/v1/solve` readiness requires API-specific validation"

- Unsafe: "billing-ready"
- Safe: "billing readiness requires billing-specific implementation and validation"

## Replace fallback language with absence language

- Unsafe: "provider fallback is supported"
- Safe: "provider fallback is not established by this packet"

- Unsafe: "hosted fallback is available"
- Safe: "hosted fallback is not established by this packet"

## Replace promotion language with preservation language

- Unsafe: "Level 2 and Level 3 can be used as promotional proof"
- Safe: "Level 2 and Level 3 remain preserved local-only, non-promotional evidence"

- Unsafe: "This evidence proves release readiness"
- Safe: "This evidence is bounded to its original packet and does not authorize release-readiness claims"
