# Unsafe Wording

## Quality evidence without execution

Unsafe wording includes any claim that implies quality evidence without executed evaluation artifacts:

- "Alpha Solver quality is validated."
- "The system produces high-quality answers."
- "Quality has been proven by Level 2 or Level 3."
- "The eval shows Alpha is accurate."
- "This packet confirms answer quality."

Use safe wording instead: "This docs-only packet defines quality-eval claim boundaries and does not establish quality evidence."

## Benchmark evidence without benchmark execution

Unsafe wording includes any claim that implies benchmark evidence without benchmark execution:

- "Alpha has benchmark evidence."
- "Alpha passed the benchmark."
- "Benchmark results show Alpha performs well."
- "Level 3 is enough benchmark evidence."
- "The system is benchmark-ready" when no benchmark readiness gate was executed.

Use safe wording instead: "No benchmark was run; benchmark evidence remains unavailable."

## Alpha superiority without comparative methodology

Unsafe wording includes any claim that implies Alpha superiority without comparative methodology:

- "Alpha is better than plain prompting."
- "Alpha outperforms other solvers."
- "Alpha beats competitors."
- "Alpha is superior for quality."
- "Level 3 proves Alpha is better."

Use safe wording instead: "No comparative methodology was executed; Alpha superiority is not established."

## Readiness and production claims without readiness gates

Unsafe wording includes claims that imply product readiness, MVP readiness, production readiness, dashboard readiness, `/v1/solve` readiness, billing readiness, provider readiness, provider fallback, hosted fallback, or evidence promotion:

- "Alpha Solver is release-ready."
- "Alpha Solver is MVP-ready."
- "Alpha Solver is production-ready."
- "The dashboard is ready."
- "`/v1/solve` is ready for users."
- "Billing is ready."
- "Hosted providers are ready."
- "Provider fallback is available."
- "Hosted fallback is available."
- "Local evidence can be promoted externally."
- "Level 2 and Level 3 authorize product claims."

Use safe wording instead: "This packet does not establish product readiness, MVP readiness, production readiness, dashboard readiness, `/v1/solve` readiness, billing readiness, provider readiness, provider fallback, hosted fallback, or evidence promotion."
