# Deferred Work

The following downstream work remains deferred:

- Level 6 product surface design;
- product UI or UX design;
- dashboard planning or route exposure;
- `/v1/solve` planning, exposure, or calls;
- provider orchestration design;
- provider fallback or hosted fallback design;
- billing planning;
- MVP readiness review;
- production readiness review;
- benchmark execution;
- model-output scoring;
- local model quality assessment.

Product surface design remains deferred until this quality evaluation design is accepted because product surfaces need a clear evidence and claim boundary before exposing user-facing flows, route assumptions, dashboard assumptions, or readiness language. Selecting Level 6 as the next lane does not start Level 6.
