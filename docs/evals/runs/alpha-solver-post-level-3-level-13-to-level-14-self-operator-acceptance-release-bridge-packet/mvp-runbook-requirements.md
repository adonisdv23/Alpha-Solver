# MVP runbook requirements

The future MVP runbook must include:

- prerequisites;
- allowed local actions;
- operator confirmation flow;
- stop states;
- artifact locations;
- redaction rules;
- dry-run procedure;
- acceptance procedure;
- blocked actions;
- rollback/abort language;
- evidence boundaries.

The runbook must be usable by an operator without implying autonomous operation, provider access, hosted fallback, deployment, billing, or source-artifact mutation.
