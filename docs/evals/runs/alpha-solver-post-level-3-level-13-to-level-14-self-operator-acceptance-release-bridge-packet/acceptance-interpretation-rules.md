# Acceptance interpretation rules

Future interpretation must apply these rules:

- acceptance evidence is local-only run evidence;
- it does not prove production readiness;
- it does not prove hosted behavior;
- it does not prove provider behavior;
- it does not prove benchmark superiority;
- it does not prove autonomous operation;
- failures must produce one selected next lane;
- partial passes must not be inflated into release readiness.

Interpretation must separate observed local run outcomes from release claims. Any unresolved blocker must prevent MVP readiness claims until a follow-up lane resolves it.
