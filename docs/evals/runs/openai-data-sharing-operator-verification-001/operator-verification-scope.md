# Operator verification scope

This packet is limited to docs-only operator-verification preparation for OpenAI data-sharing, project, billing, redaction, and go/no-go boundaries.

Allowed status values:

- `pending_operator_verification`
- `operator_verified`
- `blocked`
- `not_applicable`

Default status for all operator-account-specific items: `pending_operator_verification`.

This lane may provide checklists and intake templates. It may not assert that OpenAI account, project, data-sharing, billing, or cost settings are currently correct unless current operator evidence is committed in this packet.
