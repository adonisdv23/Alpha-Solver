# Routing Matrix

Each row is an explainable routing hypothesis for future local-only experimentation. Do not claim routing success until executed evidence exists.

| Task category | Selected route or role | Rejected routes or roles | Reason selected | Evidence needed to justify the route | Uncertainty level | Stop condition | Human escalation trigger | What must not be claimed from this route |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| Coding | Router → code reviewer → first-pass solver → critic → final synthesizer | Safety-only route unless boundary signal exists; summarizer-only route; judge-only route | Code tasks need repo-fit, tests, and implementation constraints before synthesis. | Task bank item with code signals, expected code route, model ids, prompt/output logs, reviewer checklist, test plan. | High until local code prompts are executed. | Missing files/specs, unsafe repo operation, prompt echo, empty output, non-loopback endpoint, hosted key present. | Behavior-changing patch without spec, failing tests not understood, security-sensitive code, broad refactor pressure. | Code correctness, merged-ready patch quality, benchmark performance, Alpha superiority. |
| False premise | Router → critic → final synthesizer | Direct first-pass-only route; summarizer-only route; judge-only route | False-premise tasks need premise checking before answer generation. | Prompts with known false premises, expected correction behavior, artifact showing no fabrication. | High until gold examples are run. | Model accepts false premise, fabricates citations, or cannot state uncertainty. | High-stakes factual domain, legal/medical/financial claims, source unavailable but required. | Fact-checking reliability or hallucination immunity. |
| Hidden constraints | Router → planning → first-pass solver or clarifier → final synthesizer | Immediate final solver; judge candidate; code reviewer unless code signals exist | Missing constraints require assumptions or clarifying path before substantive answer. | Task bank with missing fields, expected ask/assume behavior, route rationale. | Medium-high. | Model invents constraints, ignores ambiguity, or asks irrelevant questions. | Operator must choose business, legal, safety, or user-specific assumptions. | That Alpha can infer private or unstated requirements. |
| Safety/boundary | Router → safety reviewer → final synthesizer | Direct solver; code reviewer; judge candidate as sole reviewer | Boundary tasks need fail-closed review before any helpful content. | Gold boundary prompts, expected refusal/limited-help labels, no private data, fail-closed reason codes. | High. | Unsafe compliance, credential/API-token request, private data exposure risk, hosted key present. | Any request involving real harm, secrets, regulated advice, or private data. | Safety validation, policy completeness, production readiness. |
| Summarization | Router → first-pass summarizer → critic → final synthesizer | Code reviewer unless code content; safety reviewer unless boundary signal; judge-only | Summaries need fidelity and claim-preservation checks. | Source text hash, summary output, diff/fidelity checklist, no-new-claims review. | Medium. | Summary adds unsupported claims, omits evidence boundary, or changes verdict. | Legal/compliance summaries, private documents, disputed claims. | That summaries are lossless or legally reliable. |
| Strategy | Router → first-pass solver → critic → final synthesizer | Classification-only; code reviewer; safety-only unless boundary signal | Strategy requires options, tradeoffs, and uncertainty review. | Scenario prompt, criteria checklist, rejected options, uncertainty notes. | Medium-high. | Model gives unsupported ranking, hides uncertainty, or recommends irreversible action without evidence. | Operator decision has financial, legal, safety, or staffing consequences. | Business success, optimality, or recommendation authority. |
| Evidence audit | Router → evidence audit critic → final synthesizer | Direct solver; summarizer-only; judge-only route | Audit tasks must compare claims to artifacts before synthesis. | Claim list, source files/logs, citation map, unsupported-claim register. | Medium. | Missing artifacts, uncited claims, conflict between artifacts, overclaim pressure. | Claim affects release readiness, public claims, benchmark/value assertions. | Validation, readiness, superiority, or benchmark evidence. |
| Planning | Router → planner → critic → final synthesizer | Direct implementation route; judge-only route | Planning needs dependency, stop-condition, and validation sequencing. | Lane instructions, dependencies, proposed steps, explicit non-actions. | Medium. | Dependency missing, user authorization missing, broad eval requested, model-call precondition unmet. | Scope conflict, missing operator authorization, private-data risk. | That planned work has been executed. |
| Synthesis | Router → final synthesizer with critic review | First-pass-only; code reviewer unless code; judge candidate as sole synthesizer | Synthesis combines artifacts and must preserve boundaries. | Input artifact list, claim map, consistency check, final caveats. | Medium. | Inconsistent source packets, missing verdict, unsupported unifying claim. | Public-facing claims, conflicting evidence, high-stakes domains. | New evidence, value proof, or resolved contradictions. |
| Classification | Router/classifier → final synthesizer if explanation needed | Full council by default; code reviewer; safety reviewer unless boundary signal | Classification can be small and deterministic when labels are frozen. | Label schema, gold labels, confusion log, uncertainty threshold. | Medium-high until executed. | Label outside schema, low confidence, ambiguous prompt, high-impact label. | Classification affects access, safety, user rights, or release gates. | Reliable automation or production classifier readiness. |
