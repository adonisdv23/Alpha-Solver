# Operator runbook

Lane: `ALPHA-SOLVER-ROUTED-VS-PLAIN-PILOT-OUTPUT-COLLECTION-PREP-001`

Use this runbook only after a future operator decision separately authorizes output collection or provides outputs. This repository packet does not authorize Codex, the collector, or any agent to create outputs.

## Collection prerequisites

1. Confirm the authorized task IDs remain exactly `RVP-001` through `RVP-012`.
2. Confirm the collector has operator-provided outputs or explicit separate authorization for the execution source.
3. Confirm the collection source label, timestamp or run label, and artifact path/source reference are available before filling templates.
4. Stop if the only available outputs are generated by Codex without operator-provided provenance or separate execution authorization.

## Collection steps

1. Copy the blank fields from `task-output-capture-template.md` for each task.
2. Paste operator-provided plain baseline output and routed Alpha output exactly as supplied or reference an operator-held artifact path.
3. Copy route metadata from `route-metadata-capture-template.md` only when operator-provided or separately authorized metadata exists.
4. Preserve task IDs exactly.
5. Mark sensitive/private data confirmation before committing any filled template.
6. Do not score, compare, rank, unblind, or interpret outputs in the collection packet.

## Stop conditions

- No operator-provided output evidence exists.
- Provider, hosted-model, local-model, tool, web, or `/v1/solve` execution would be needed but has not been separately authorized.
- A task involves high-stakes content and operator instructions do not specify the safe capture treatment.
- Sensitive or private data appears in supplied artifacts and redaction/custody is unresolved.
