# High-stakes card handling

High-stakes tasks require capture-only treatment unless a later authorization says otherwise. The collector must not provide advice, complete missing answers, expand outputs, score outputs, or normalize unsafe content.

If an operator-provided output contains medical, legal, financial, safety, self-harm, security, privacy, or other sensitive/high-stakes content, preserve the evidence boundary and follow the operator's redaction/custody instructions before committing any text.

Stop rather than inventing or completing high-stakes output fields when provenance, consent, redaction, or authorization is unclear.
