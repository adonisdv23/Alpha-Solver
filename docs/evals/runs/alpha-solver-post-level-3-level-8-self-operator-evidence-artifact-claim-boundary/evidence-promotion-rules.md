# Evidence Promotion Rules

## Default state

All future Self Operator artifacts remain non-promotional unless a later approved lane explicitly promotes them. This packet itself does not promote evidence.

## Promotion prerequisites

A future evidence-promotion lane must, at minimum:

- Name the source artifacts being considered.
- Preserve each source artifact's original evidence boundary.
- Define the exact claim being promoted.
- Identify the reviewer authority for promotion.
- Record checks, failure conditions, and unresolved caveats.
- State which blocked claims remain blocked after promotion.

## Promotion limits

Promotion of one narrow claim does not imply promotion of adjacent claims. For example, acceptance of a reviewed operator note does not imply autonomous operation, production readiness, provider readiness, model quality, benchmark performance, deployment readiness, browser automation readiness, billing readiness, or external-action readiness.

## Rejection default

If source provenance, reviewer authority, scope, or claim language is incomplete, evidence promotion is rejected by default.
