# Readiness Claim Rules

## Readiness claims are blocked by default

Self Operator readiness claims are blocked unless a future approved lane names the exact readiness dimension, evidence, scope, checks, reviewer authority, and residual caveats.

## Required readiness dimensions

A future readiness claim must be scoped to a single dimension. It must not combine or imply any of the following blocked dimensions without separate approval:

- Autonomous operation.
- Production readiness.
- Provider readiness.
- Model quality.
- Benchmark performance.
- Deployment readiness.
- Browser automation readiness.
- Billing readiness.
- External-action readiness.

## Required wording

Acceptable wording must describe artifact status rather than readiness. For example, use "artifact reviewed for boundary completeness" instead of "Self Operator is ready."

## Contradiction handling

If a readiness statement conflicts with this packet, `blocked-claims.md` controls and the readiness statement is blocked.
