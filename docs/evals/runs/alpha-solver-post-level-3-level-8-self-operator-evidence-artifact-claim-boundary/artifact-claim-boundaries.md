# Artifact Claim Boundaries

## Purpose

Future Self Operator artifacts may be useful as preserved records, but artifact existence is not equivalent to readiness. This file defines the maximum claims those artifacts can support unless an approved future lane explicitly promotes evidence.

## Claims future artifacts can support

A future artifact may support claims that it:

- Exists at a specific repository path.
- Was produced by a named lane or packet.
- Contains the preserved text, decisions, screenshots, transcripts, logs, templates, or reviewer notes recorded in that artifact.
- Was reviewed under a stated docs-only or run-specific review process.
- Has not been promoted beyond its stated evidence boundary.

## Claims future artifacts cannot support by default

A future artifact cannot prove:

- Autonomous operation.
- Production readiness.
- Provider readiness.
- Model quality.
- Benchmark performance.
- Deployment readiness.
- Browser automation readiness.
- Billing readiness.
- External-action readiness.

## Required framing

Any future use of Self Operator artifacts must state whether the artifact is raw, reviewed, accepted for a narrow purpose, or promoted by a later approved lane. Ambiguous claims must default to blocked.
