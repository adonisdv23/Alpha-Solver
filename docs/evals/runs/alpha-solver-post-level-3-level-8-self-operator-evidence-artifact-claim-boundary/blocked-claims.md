# Blocked Claims

This packet blocks the following claim categories for future Self Operator artifacts unless a later approved lane explicitly promotes evidence under a stronger review process.

## Blocked readiness and capability claims

Future Self Operator artifacts must not be cited as proof of:

- Autonomous operation.
- Production readiness.
- Provider readiness.
- Model quality.
- Benchmark performance.
- Deployment readiness.
- Browser automation readiness.
- Billing readiness.
- External-action readiness.

## Blocked inference patterns

The following inference patterns are also blocked:

- Treating artifact presence as evidence of operational capability.
- Treating a docs-only review as a runtime validation.
- Treating a local or bounded observation as a generalized benchmark.
- Treating an accepted artifact as approval to deploy, bill, call providers, automate browsers, or perform external actions.
- Treating absence of a recorded failure as evidence of readiness.

## Default rule

If a claim could be read as one of the blocked categories above, it must be rewritten, downgraded, or rejected.
