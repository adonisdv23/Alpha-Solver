# UI Copy Boundaries

## UI copy limit

Future UI copy must be limited to factual, bounded, non-promotional statements supported by accepted evidence for the exact surface. This packet does not implement UI copy and does not authorize any UI text to be exposed to users.

## Prohibited UI copy

Future UI copy must not state or imply:

- quality validation;
- benchmark performance;
- Alpha superiority;
- MVP readiness;
- production readiness;
- dashboard readiness;
- `/v1/solve` readiness;
- provider readiness;
- billing readiness;
- fallback readiness;
- public launch readiness;
- evidence promotion.

## Required UI copy qualifiers

If a later authorized lane permits UI copy, the copy must include sufficient qualifiers to prevent overclaiming. At minimum, future UI copy must identify whether a feature is experimental, local-only, internal-only, unavailable, gated, or otherwise limited.

## UI stop rule

If UI copy cannot be written without implying a blocked claim, the future lane must stop and use the appropriate fallback path. Level 6 controls whether and how this packet is used for that decision.
