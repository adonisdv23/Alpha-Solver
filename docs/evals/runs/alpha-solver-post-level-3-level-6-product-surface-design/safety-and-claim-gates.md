# Safety and Claim Gates

## Safety gates

Future product-surface work must prove the following before implementation:

- product-surface exposure is disabled by default;
- external and hosted behavior is disabled by default;
- operator opt-in is explicit and auditable;
- missing controls fail closed;
- claim-boundary copy is present before any operator-facing surface appears;
- redaction and sensitive-data handling are defined;
- evidence promotion is impossible without a separate accepted lane;
- stop conditions are implemented or otherwise enforceable by process.

## Claim boundaries

This packet permits only claims that a docs-only Level 6 design packet was authored and checked. It does not support claims of:

- API readiness;
- dashboard readiness;
- product readiness;
- MVP readiness;
- production readiness;
- provider readiness;
- billing readiness;
- model quality;
- benchmark performance;
- Alpha superiority;
- customer usability.

## Required wording boundary

Future surfaces must use wording that identifies evidence as local, design-only, non-promotional, and non-product-ready unless a later accepted evidence packet expands that boundary.
