# Release Closeout Checklist

Future release closeout must verify:

- [ ] Acceptance prerequisites were met before any run.
- [ ] Acceptance evidence exists in an allowed local-only packet.
- [ ] Evidence has been interpreted under the evidence boundary.
- [ ] Defects and caveats are documented.
- [ ] Blocked claims remain blocked.
- [ ] Release notes do not overclaim MVP, runtime, production, provider, hosted, benchmark, autonomous, deployment, billing, or external-action readiness.
- [ ] ED handoff is separate from GS handoff.
- [ ] GS update, if any, is performed by an authorized operator outside this packet.
- [ ] Selected next action or fallback lane is documented.
