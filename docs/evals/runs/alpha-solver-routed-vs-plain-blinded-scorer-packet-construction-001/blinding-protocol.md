# Blinding protocol

Scorer-facing materials must not reveal whether Response A or Response B came from the plain or routed source arm.

Rules applied:

- use only `Response A` and `Response B` labels;
- remove source-identifying headings and source labels;
- remove route labels and routed-only boundary scaffolding from scorer-facing response text;
- include source-neutral task context from committed task cards;
- do not include route metadata identity, model identity, source paths, source folder names, or an A/B key;
- keep all scoring, preference, rationale, winner, aggregate, and interpretation fields blank.

Any future unblinding requires a separately authorized lane.
