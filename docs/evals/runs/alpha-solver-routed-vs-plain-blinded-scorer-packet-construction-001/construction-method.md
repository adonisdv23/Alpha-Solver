# Construction method

The packet was constructed from committed routed-vs-plain manual output collection artifacts and the committed pilot task set. Source outputs were transformed into scorer-facing Response A and Response B fields with source-identifying labels, route labels, Alpha/plain labels, model identity labels, and routed-only boundary scaffolding removed from scorer-facing text.

Task context was copied from the committed source-neutral task cards so scorers can evaluate responses without opening source output folders, route metadata files, or A/B key material.

No source output was changed. No provider, hosted model, local model, runtime endpoint, tool, browser, Google Sheet, dependency, dashboard, or public API was used.
