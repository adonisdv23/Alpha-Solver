# Forbidden first-code files

A future first-code lane must not modify these areas.

## Forbidden until separate explicit runtime lane

- Runtime source files.
- Provider source files.
- API route files.
- Dashboard files.
- CLI files.
- Checker scripts.
- `Makefile`.
- CI workflows.

## Must never be modified by Self Operator lanes without separate authorization

- Source-artifact files and directories.
- Existing evidence packets.
- Existing Level 8 packets.
- Existing Level 9 packets.
- Backlog workbooks or external planning ledgers.
- Files containing credentials, secrets, billing records, browser profiles, deployment output, or provider response payloads.
