# Allowed modification scope

This packet itself allows only docs under its own new packet directory.

For a future first-code lane, the anticipated allowed modification scope is limited to:

- Static test scaffold files under a future `tests/self_operator/` area.
- Inert synthetic fixtures under a future `tests/fixtures/self_operator/` area.
- Review notes created by that future lane if explicitly allowed by its controlling packet.

The future lane must stop if changed files exceed its allowed scope.
