# Alpha Solver Source-Pack Surface Map

Lane: `ALPHA_SOLVER_SOURCE_PACK_SURFACE_MAP_DOCS_MVP-001`  
Issue: #641

## Scope and non-goals

This document is a docs-only inspection aid for future Council/Alpha coordination. It records high-level Alpha-Solver-native surfaces that may matter to a future source-pack construction discussion, based only on bounded repository guidance, indexes, metadata, and file/directory names.

This document does not:

- implement a source-pack compiler;
- build a research manifest index;
- change retrieval, vector, graph, database, runtime, scoring, unblinding, provider, model, API, browser, or LLM behavior;
- authorize future implementation;
- make readiness, production, release, benchmark, provider-validation, autonomous-readiness, model-superiority, runtime/scoring/unblinding, final automated MVP, or unbounded final automated MVP claims.

## Evidence boundary

The map is intentionally conservative. It classifies surfaces by repository guidance, README/index files, explicit metadata files, and file or directory names only. It does not inspect arbitrary implementation source bodies, dataset contents, generated outputs, provider outputs, raw run artifacts, secrets, credentials, private data, or unrelated application internals. A future source-pack effort would need its own approved scope, audit criteria, and implementation authorization.

## Future audit cautions

- Reconfirm the baseline branch, issue state, and current repo guidance before any future source-pack work.
- Establish whether each candidate artifact is canonical source, generated derivative, config, test fixture, example, or runtime-adjacent before inclusion.
- Review licensing, privacy, secrets, generated-output, raw-artifact, and provider-output boundaries before reading or packaging data-like surfaces.
- Keep source-pack mapping separate from implementation, runtime integration, manifest indexing, retrieval/vector/graph/database work, scoring, unblinding, model/provider validation, and readiness claims unless a future lane explicitly authorizes that work.
